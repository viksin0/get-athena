VS Code Extension · v0.1.7 (preview)

Athena

A browser-style UI for the Athena AI design pipeline. Turn a sentence into a reviewable Azure-portal blade — research, ideate, generate, validate — all inside VS Code.

↓ Download extension athena-0.1.7.vsix · 392 KB Read the install guide
Before you download — this is not a standalone extension. Athena drives a fleet of AI agents that live inside the Athena starter workspace and are executed by GitHub Copilot Chat in Agent Mode. You need that workspace open in VS Code and an active Copilot subscription for anything to run. The workspace repo is private, so you'll need access granted before you can clone it. The checklist below walks you through all of it.

What you need first

Confirm all four before installing — the extension will look broken without them.

RequirementWhy
VS Code 1.92 or newerThe extension targets this engine version.
GitHub Copilot + Copilot Chat extensions, signed inAgents run on your Copilot language models — no separate API key needed.
Copilot Agent Mode availableThe @athena agent can only spawn @design, @research, @reflect, and @figma when Chat is in Agent Mode.
The Athena starter workspace, cloned locallyThe agents, design rules, and the designs/ folder all live in this repo. The extension reads them from your open workspace.

1Install & run

  1. Get access to the workspace

    The Athena starter repo is private. First, request access from the person who shared this page (or the repo owner). You'll know you have it when github.com/viksin0/athena-starter opens instead of showing a 404.

    Once you have access, clone it and open it in VS Code:

    git clone https://github.com/viksin0/athena-starter.git
cd athena-starter
code .

    Then, inside VS Code, install workspace dependencies once:

    npm install

  2. Download the extension

    Grab athena-0.1.7.vsix (the button at the top of this page).

  3. Install the VSIX

    Two equivalent ways:

    • UI: Extensions view (⇧⌘X / Ctrl+Shift+X) → ··· menu → Install from VSIX… → pick the downloaded file.
    • Terminal: code --install-extension athena-0.1.7.vsix

  4. Reload the window

    ⌘⇧P / Ctrl+Shift+P → Developer: Reload Window. The Athena icon appears in the Activity Bar and a local bridge starts on localhost:7878.

  5. Open Athena

    Click the Athena icon in the Activity Bar, then Open in VS Code Panel (or Open in Browser). The Get Started page loads automatically the first time.

2Switch Copilot Chat to Agent Mode

This is the step people forget. Open Copilot Chat and change the mode picker at the top from Ask to Agent. If you run a stage while Chat is in Ask mode, @athena cannot delegate to the rest of the fleet and the run silently does nothing.

3Run the agents

From the Athena panel you can drive the fleet three ways:

Create a design

Click ＋ Create new design, describe the blade in a sentence, and pick an outcome.

Run a stage

Use the stage stepper (Research → Ideation → Generation → Validation → Learning) to fire one stage.

Run everything

Hit ▶ Run Auto to chain all five stages back-to-back.

You can also type free-form chat or a slash command (e.g. /research, /generate) directly in the Athena chat. Responses stream back into the panel.

Full documentation lives in the built-in 📖 Get-Started Docs (the book icon in the panel) — the same content as the Playground/Get started/ handbook.

Troubleshooting

SymptomFix
Panel is empty / "No designs found"You didn't open the athena-starter folder as your workspace. Open the folder that contains designs/.
Running a stage does nothingCopilot Chat is in Ask mode. Switch it to Agent (section 2).
"Open in Browser" shows Not foundA stale window is holding port 7878. Reload the VS Code window once.
Model errors / not signed inConfirm the GitHub Copilot and Copilot Chat extensions are installed and you're signed in with an active subscription.
